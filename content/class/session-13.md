## 1. 今日目標不是「新功能」，是「敢不敢交差」

比賽前最常見失敗：

- 連結點了 404  
- 只有本機好看、正式建置失敗  
- 路徑改名導致評審找不到  
- 內容看起來完整但其實是編的

## 3. 完整步驟：`npm run build`

### 步驟 1
停掉 dev 也可以，另開終端機視窗較不易混。

### 步驟 2

```bash
cd 你的/iGemSite
npm run build
```

### 步驟 3：成功

看到類似 Compiled / Generating static pages 且最後沒有 `Failed`。  
截圖保存。

### 步驟 4：失敗

1. 不要慌，往上找第一個 `Error`  
2. 通常會寫檔案路徑與行號  
3. 常見類型：  
   - TypeScript 型別  
   - 找不到模組  
   - md 對應 slug 缺失  
4. 修完再 build，直到過或記錄「卡關點」請老師

### 步驟 5：建置與開發的差別（必懂）

| | `npm run dev` | `npm run build` |
|--|---------------|-----------------|
| 用途 | 邊改邊看 | 模擬正式輸出 |
| 嚴格度 | 較寬容 | 較嚴格 |
| 課堂意義 | 日常 | 交件前必做 |

## 4. 上線／Freeze 概念課（30 分）

請閱讀並討論 `docs/DEPLOY_TO_IGEM.md` 大意：

1. 本機或 GitHub Pages ≠ 自動等於競賽封存位址  
2. 最終需符合 iGEM 托管與資源規則（字型、圖、外連）  
3. Wiki Freeze 後不能大改  
4. 內容仍以團隊確認與官方要求為準  

學生不需今天就完成部署，但要能**說出差別**。
